進めます。まずは **サーバーなし・端末内で指定時刻に鳴るローカルリマインダー** として実装します。`flutter_local_notifications` は現在 `22.0.1` が出ており、スケジュール通知には `timezone` / `zonedSchedule` を使う形です。([Dart packages][1])

Android はここが罠です。通常のリマインダーなら、まずは **exact alarm を使わず `inexactAllowWhileIdle`** にします。Android 14 以降では exact alarm 権限が初期状態で許可されないケースがあり、正確な時刻通知を前提にすると権限導線と審査リスクが増えます。([Android Developers][2])

## 1. 依存追加

```yaml
# pubspec.yaml
dependencies:
  flutter_local_notifications: ^22.0.1
  timezone: ^0.10.1
  flutter_timezone: ^5.1.0
```

`flutter_timezone` は端末の IANA timezone identifier を取るために使います。現行版では `FlutterTimezone.getLocalTimezone()` が `TimezoneInfo` を返します。([Dart packages][3])

## 2. Android 設定

`android/app/build.gradle` または `build.gradle.kts` に desugaring を入れます。公式 README では Java 17、`coreLibraryDesugaring`、`compileSdk 35+` が必要とされています。([Dart packages][1])

Groovy の場合:

```gradle
android {
    compileSdk 36

    defaultConfig {
        multiDexEnabled true
    }

    compileOptions {
        coreLibraryDesugaringEnabled true
        sourceCompatibility JavaVersion.VERSION_17
        targetCompatibility JavaVersion.VERSION_17
    }

    kotlinOptions {
        jvmTarget = JavaVersion.VERSION_17.toString()
    }
}

dependencies {
    coreLibraryDesugaring 'com.android.tools:desugar_jdk_libs:2.1.4'
}
```

`android/app/src/main/AndroidManifest.xml`:

```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED"/>

    <application
        android:label="your_app_name"
        android:name="${applicationName}"
        android:icon="@mipmap/ic_launcher">

        <receiver
            android:exported="false"
            android:name="com.dexterous.flutterlocalnotifications.ScheduledNotificationReceiver" />

        <receiver
            android:exported="false"
            android:name="com.dexterous.flutterlocalnotifications.ScheduledNotificationBootReceiver">
            <intent-filter>
                <action android:name="android.intent.action.BOOT_COMPLETED"/>
                <action android:name="android.intent.action.MY_PACKAGE_REPLACED"/>
                <action android:name="android.intent.action.QUICKBOOT_POWERON"/>
                <action android:name="com.htc.intent.action.QUICKBOOT_POWERON"/>
            </intent-filter>
        </receiver>

        <!-- 既存の activity はそのまま -->
    </application>
</manifest>
```

`flutter_local_notifications` v16 以降は最低限の権限しか plugin 側 manifest に持たないため、スケジュール通知では `RECEIVE_BOOT_COMPLETED` と receiver をアプリ側に追加する必要があります。Android 13 以降の通知許可は runtime prompt で要求します。([Dart packages][1])

## 3. iOS 設定

`ios/Runner/AppDelegate.swift` に追加します。

```swift
import UIKit
import Flutter
import UserNotifications

@main
@objc class AppDelegate: FlutterAppDelegate {
  override func application(
    _ application: UIApplication,
    didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
  ) -> Bool {
    UNUserNotificationCenter.current().delegate = self

    GeneratedPluginRegistrant.register(with: self)
    return super.application(application, didFinishLaunchingWithOptions: launchOptions)
  }
}
```

iOS はユーザー許可を明示的に取り、`IOSFlutterLocalNotificationsPlugin.requestPermissions()` で alert/sound/badge を要求します。([Dart packages][4])

## 4. 通知サービスを追加

`lib/services/reminder_notification_service.dart`

```dart
import 'package:flutter/foundation.dart';
import 'package:flutter_local_notifications/flutter_local_notifications.dart';
import 'package:flutter_timezone/flutter_timezone.dart';
import 'package:timezone/data/latest.dart' as tz_data;
import 'package:timezone/timezone.dart' as tz;

class ReminderNotificationService {
  ReminderNotificationService._();

  static final ReminderNotificationService instance =
      ReminderNotificationService._();

  final FlutterLocalNotificationsPlugin _plugin =
      FlutterLocalNotificationsPlugin();

  static const String _channelId = 'reminders';
  static const String _channelName = 'Reminders';
  static const String _channelDescription = 'User scheduled reminder notifications';

  bool _initialized = false;

  Future<void> init({
    void Function(NotificationResponse response)? onNotificationTap,
  }) async {
    if (_initialized) return;

    tz_data.initializeTimeZones();

    try {
      final timezoneInfo = await FlutterTimezone.getLocalTimezone();
      tz.setLocalLocation(tz.getLocation(timezoneInfo.identifier));
    } catch (_) {
      // 取得に失敗した場合でもクラッシュさせない。
      // Asia/Tokyo 固定ではなく UTC に落とす。アプリ側でログ送信してもよい。
      tz.setLocalLocation(tz.UTC);
    }

    const androidSettings = AndroidInitializationSettings('@mipmap/ic_launcher');

    const iosSettings = DarwinInitializationSettings(
      requestAlertPermission: false,
      requestBadgePermission: false,
      requestSoundPermission: false,
    );

    const settings = InitializationSettings(
      android: androidSettings,
      iOS: iosSettings,
    );

    await _plugin.initialize(
      settings,
      onDidReceiveNotificationResponse: onNotificationTap,
    );

    await _plugin
        .resolvePlatformSpecificImplementation<
            AndroidFlutterLocalNotificationsPlugin>()
        ?.createNotificationChannel(
          const AndroidNotificationChannel(
            _channelId,
            _channelName,
            description: _channelDescription,
            importance: Importance.high,
          ),
        );

    _initialized = true;
  }

  Future<bool> requestPermissions() async {
    final androidGranted = await _plugin
        .resolvePlatformSpecificImplementation<
            AndroidFlutterLocalNotificationsPlugin>()
        ?.requestNotificationsPermission();

    final iosGranted = await _plugin
        .resolvePlatformSpecificImplementation<
            IOSFlutterLocalNotificationsPlugin>()
        ?.requestPermissions(
          alert: true,
          badge: true,
          sound: true,
        );

    return androidGranted ?? iosGranted ?? true;
  }

  Future<void> scheduleOneShot({
    required int id,
    required String title,
    required String body,
    required DateTime scheduledAt,
    String? payload,
  }) async {
    await init();

    if (!scheduledAt.isAfter(DateTime.now())) {
      throw ArgumentError.value(
        scheduledAt,
        'scheduledAt',
        '通知予定時刻は現在時刻より後にしてください',
      );
    }

    await _plugin.zonedSchedule(
      id: id,
      title: title,
      body: body,
      scheduledDate: tz.TZDateTime.from(scheduledAt, tz.local),
      notificationDetails: _notificationDetails(),
      androidScheduleMode: AndroidScheduleMode.inexactAllowWhileIdle,
      payload: payload,
    );
  }

  Future<void> scheduleDaily({
    required int id,
    required String title,
    required String body,
    required int hour,
    required int minute,
    String? payload,
  }) async {
    await init();

    final now = tz.TZDateTime.now(tz.local);
    var firstSchedule = tz.TZDateTime(
      tz.local,
      now.year,
      now.month,
      now.day,
      hour,
      minute,
    );

    if (!firstSchedule.isAfter(now)) {
      firstSchedule = firstSchedule.add(const Duration(days: 1));
    }

    await _plugin.zonedSchedule(
      id: id,
      title: title,
      body: body,
      scheduledDate: firstSchedule,
      notificationDetails: _notificationDetails(),
      androidScheduleMode: AndroidScheduleMode.inexactAllowWhileIdle,
      matchDateTimeComponents: DateTimeComponents.time,
      payload: payload,
    );
  }

  Future<void> cancel(int id) async {
    await _plugin.cancel(id: id);
  }

  Future<void> cancelAll() async {
    await _plugin.cancelAll();
  }

  Future<List<PendingNotificationRequest>> pending() {
    return _plugin.pendingNotificationRequests();
  }

  NotificationDetails _notificationDetails() {
    return const NotificationDetails(
      android: AndroidNotificationDetails(
        _channelId,
        _channelName,
        channelDescription: _channelDescription,
        importance: Importance.high,
        priority: Priority.high,
      ),
      iOS: DarwinNotificationDetails(
        presentAlert: true,
        presentBadge: true,
        presentSound: true,
      ),
    );
  }
}
```

Future<void> main() async {

## 6. 使用例

通知許可は、アプリ起動直後ではなく「リマインダーを有効にする」操作時に出す方が自然です。

```dart
Future<void> enableReminder() async {
  final granted =
      await ReminderNotificationService.instance.requestPermissions();

  if (!granted) {
    // UI 側で「設定から通知を許可してください」を出す
    return;
  }

  await ReminderNotificationService.instance.scheduleOneShot(
    id: 1001,
    title: 'リマインダー',
    body: 'そろそろ確認する時間です',
    scheduledAt: DateTime.now().add(const Duration(minutes: 10)),
    payload: 'reminder:1001',
  );
}
```

毎日 9:00 のリマインダーなら:

```dart
await ReminderNotificationService.instance.scheduleDaily(
  id: 2001,
  title: '今日のリマインダー',
  body: '今日のタスクを確認しましょう',
  hour: 9,
  minute: 0,
  payload: 'daily-reminder',
);
```

## この実装でまだ決めていない点

この実装は「だいたい指定時刻に出ればよい」リマインダーです。服薬・アラーム・カレンダーのように **秒単位または分単位で正確に鳴らないと価値が壊れる** なら、`AndroidScheduleMode.exactAllowWhileIdle` と exact alarm 権限導線を別途入れるべきです。ただし Android 側では special access が絡み、`SCHEDULE_EXACT_ALARM` / `USE_EXACT_ALARM` の扱いとストア審査を設計に含める必要があります。([Android Developers][2])

また、iOS には pending local notifications の保持数制限があるため、大量の未来通知を全部登録する設計は避けた方がよいです。例えば「次の 30 件だけ登録して、アプリ起動時に補充する」方式が無難です。([Dart packages][1])

こちらではまだリポジトリに直接パッチを当てていません。次に必要なのは `pubspec.yaml`、`android/app/build.gradle` または `build.gradle.kts`、`AndroidManifest.xml`、`AppDelegate.swift`、通知を呼び出す画面の該当部分です。
